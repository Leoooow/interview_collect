## 1. Agent 核心技术

### 1.1 Agent 基础架构

#### Q1: 请介绍一下 Agent 的核心组成模块？

**参考要点：**
- **感知模块（Perception）**：接收用户输入，理解意图
- **规划模块（Planning）**：任务分解、路径规划、反思修正
- **记忆模块（Memory）**：短期记忆（上下文）、长期记忆（向量库）
- **工具调用（Tool Use）**：API调用、函数执行、外部系统交互
- **行动模块（Action）**：执行具体操作，返回结果

**可能追问：**
- 你在荣耀的项目中，哪个模块最复杂？如何解决的？
- 蚂蚁阿福的健康问答场景，哪些模块最重要？

#### Q2: ReAct 框架的原理是什么？你如何应用的？

**参考要点：**
- **ReAct = Reasoning + Acting**
- 循环流程：Thought（思考）→ Action（行动）→ Observation（观察）→ Thought...
- 相比纯 Chain-of-Thought，增加了与环境的交互能力

**简历关联：**
- 在客服Agent中如何使用ReAct处理复杂查询？
- 如何优化 Thought 的提示词来提升推理质量？

#### Q3: 如何处理 Agent 的"循环推理"或"死循环"问题？

**参考要点：**
- 设置最大迭代次数限制
- 检测重复的 Action/Thought
- 设计终止条件（满足目标、检测到无意义的循环）
- 引入"自我反思"机制判断何时停止

### 1.2 规划与决策

#### Q4: 如何设计一个复杂任务的规划流程？

**参考要点：**
- **任务分解**：将复杂目标拆解为子任务（Task Decomposition）
- **规划算法**：
  - 静态规划：Plan-and-Execute（先规划后执行）
  - 动态规划：ReAct、Task Weaving
- **优先级管理**：依赖关系图、任务队列
- **动态调整**：执行过程中根据反馈调整计划

**简历关联：**

- **状态传递**：每个节点更新共享状态
- **条件边**：基于状态动态选择下一个节点
- **循环与递归**：支持复杂的控制流

**简历关联：**

## 2. RAG 检索增强生成

### 2.1 RAG 核心流程

#### Q10: 请描述一个完整的 RAG 链路，并说明每个环节的优化点？

**参考要点：**

| 环节 | 核心功能 | 优化策略 |
|------|----------|----------|
| **文档解析** | 提取文本、结构化 | 多模态解析（PDF、表格、图片）、OCR增强 |
| **分块（Chunking）** | 切分文档 | 语义分块、滑动窗口、元数据保留 |
| **嵌入（Embedding）** | 向量化 | 选择合适的 Embedding 模型、多向量代表 |
| **索引构建** | 加速检索 | 向量索引（HNSW、IVF）、混合索引 |
| **检索（Retrieval）** | 召回相关文档 | 相似度检索、混合检索（向量+关键词）、重排序 |
| **生成（Generation）** | LLM 生成答案 | Prompt 优化、引用标注、答案验证 |

**简历关联：**
- 你在荣耀如何优化客服场景的 RAG 召回率？
- 如何通过检索策略优化将关键指标提升 10% 的？

#### Q11: 向量检索和关键词检索如何结合（混合检索）？

**参考要点：**
- **倒排索引（BM25）**：精确匹配关键词
- **向量检索**：语义相似度匹配
- **融合策略**：
  - 加权融合（Reciprocal Rank Fusion）
  - 学习型排序（Learning to Rank）
- **适用场景**：
  - 关键词检索：专业术语、编号、精确名称
  - 向量检索：语义理解、同义词、概念匹配

**业务关联：**
- 蚂蚁阿福的医疗问答中，如何处理疾病名称、药品名称的精确匹配？

### 2.2 检索优化

#### Q12: 如何提升 RAG 的召回率？

**参考要点：**
- **Query 改写**：
  - 查询扩展（同义词、上位词、下位词）
  - 多路径查询（生成多个不同角度的 query）
  - HyDE（假设答案，反向检索）
- **索引优化**：
  - 多粒度分块（句子、段落、文档级）
  - 元数据过滤
- **检索策略**：
  - Top-K 动态调整
  - 跨索引检索
  - 递归检索（先检索概要，再深入细节）

**简历关联：**
- 你在荣耀如何通过"分级检索策略"降低 40% 延迟的？

#### Q13: 什么是 Rerank（重排序）？为什么需要它？

**参考要点：**
- **问题**：向量检索的语义相似度≡≠答案相关性
- **Rerank 模型**：
  - 交叉编码器（Cross-Encoder）：精排 query-document 相关性
  - 模型选择：BGE-Reranker、Cohere Rerank、Cross-Encoder
- **优化策略**：
  - 召回（Retrieval）→ 粗排（Top 50-100）→ 精排（Rerank Top 10-20）
  - 两阶段架构平衡速度与效果

### 2.4 高级 RAG

#### Q15: 如何处理多轮对话中的 RAG 上下文管理？

**参考要点：**
- **Query Rewriting**：将当前问题结合历史上下文改写为独立查询
- **上下文压缩**：对检索到的文档进行去重、合并、提取关键信息
- **动态检索**：根据对话进展判断是否需要重新检索
- **引用管理**：标注答案来源，支持追溯

**业务关联：**
- 蚂蚁阿福的 AI 诊室场景，用户分多次描述症状，如何拼接完整病历？

#### Q16: Agent RAG 和普通 RAG 的区别是什么？

**参考要点：**
- **Agent RAG**：主动检索，根据任务动态决策检索时机和内容
- **普通 RAG**：被动检索，每次 query 都固定检索
- **Agent 优势**：
  - 多步推理，分阶段检索
  - 工具调用结合（查询数据库、API）
  - 自我反思，检索失败时调整策略

## 3. 大模型训练与微调

### 3.1 微调范式

#### Q17: SFT、LoRA、P-Tuning 的区别和选择？

**参考要点：**

| 方法 | 原理 | 优点 | 缺点 | 适用场景 |
|------|------|------|------|----------|
| **Full Fine-tuning (SFT)** | 更新全部参数 | 效果最好 | 显存占用大、成本高 | 数据充足、追求极致效果 |
| **LoRA** | 低秩分解，冻结主干 | 显存友好、可插拔 | 效果略低于全量微调 | 快速迭代、多任务场景 |
| **P-Tuning** | Prompt 连续化表示 | 稳定性好 | 能力上限较低 | 小模型、特定任务 |

**简历关联：**
- 你在荣耀的微调平台如何支持 SFT 和 LoRA 的？

#### Q18: 如何构造高质量的 SFT 数据集？

**参考要点：**
- **数据来源**：
  - 真实用户日志（脱敏）
  - 人工构造（专家编写）
  - 合成数据（强模型生成，弱模型筛选）
- **数据质量**：
  - 多样性（覆盖不同场景、难度）
  - 准确性（人工审核或自动验证）
  - 平衡性（正负样本均衡、长尾分布处理）
- **数据格式**：
  - Instruction-Input-Output 三元组
  - 对话历史（多轮场景）
  - 思维链（CoT，增强推理能力）

### 4.2 评测方法

#### Q23: 自动化评测和人工评测如何结合？

**参考要点：**
- **自动化评测**：
  - 单元测试（特定功能）
  - 模拟用户测试（LLM-as-a-Judge）

  - 专家评估（复杂场景）
  - 真实用户 A/B 测试
  - 标注数据集评测
- **结合策略**：
  - 自动化筛选 + 人工复核
  - 自动化监控趋势 + 人工 root cause 分析

- **多模型投票**：减少单一模型的偏好
- **与人类对齐**：定期校准 LLM 评分与人类评分的相关性

### 6.3 RAG 知识库优化

#### Q33: 你如何将检索延迟降低 40% 的？

- **查询优化**：
  - 元数据预过滤（缩小检索范围）
  - 并行检索（多路召回）
  - 结果缓存

**可能的追问：**
- 如何在降低延迟的同时保证召回率？

## 7. 蚂蚁阿福业务场景

### 7.1 健康问答

#### Q34: 蚂蚁阿福每天处理 500 万健康问题，如何保证答案的准确性和安全性？

**参考要点：**
- **知识来源**：
  - 医学文献、临床指南（权威来源）
  - 专业医生审核（人工标注）
  - 知识图谱（疾病、药品、症状关系）
- **质量保障**：
  - 多模型验证（交叉验证）
  - 置信度阈值（低置信度转人工）
  - 引用追溯（标注来源）
- **安全机制**：
  - 敏感问题拦截（涉及生命危险的，建议就医）
  - 免责声明（AI 建议不能替代医生）
  - 用户反馈机制（错误举报）

#### Q35: 用户问"我头痛怎么办？"，Agent 如何处理？

**回答流程示例：**
1. **意图识别**：健康咨询
2. **信息收集**：追问症状细节（持续时间、伴随症状、病史等）
3. **知识检索**：检索头痛相关医学知识
4. **风险判断**：是否有危险信号（如剧烈头痛、呕吐等）
5. **建议生成**：
   - 低风险：休息、饮食建议、非处方药（注明慎用）
   - 中风险：建议就医
   - 高风险：紧急就医提醒
6. **后续跟进**：提醒记录健康日记

### 7.2 AI 诊室

#### Q36: AI 诊室如何模拟医生问诊流程？

**参考要点：**
- **多轮对话设计**：
  - 引导式提问（按照主诉 → 现病史 → 既往史 → 个人史的顺序）
  - 动态调整（根据用户回答调整下一个问题）
- **病历生成**：
  - 结构化提取（症状、时间、程度）
  - 生成标准化病历描述
- **初步判断**：
  - 可能疾病列表（概率排序）
  - 建议检查项目
  - 建议就诊科室
- **边界管理**：
  - 明确告知"仅为初步判断，需医生确诊"
  - 紧急情况识别并建议立即就医

#### Q37: 如何处理用户隐私和健康数据的存储？

**参考要点：**
- **数据加密**：传输加密（TLS）、存储加密
- **权限控制**：严格的访问权限（仅用户本人）
- **数据脱敏**：日志、训练数据中去除个人身份信息
- **合规性**：符合个人信息保护法、医疗数据管理规定
